O **Interface Segregation Principle (ISP)** define que clientes não devem set forçados a depender de métodos que não utilizam, recomendando **interfaces pequenas e específicas**; em Java, isso evita contratos genéricos demais que geram implementações artificiais, como separar funções de impressão e digitalização em vez de uma interface única para todos os dispositivos, tornando o código mais coeso e flexível.

---
## Informações Adicionais Para Aprofundamento

### Exemplo De Violação
```java
public interface Maquina {
    void imprimir();
    void digitalizar();
}
```

### Exemplo Aplicando ISP
```java
public interface Impressora {
    void imprimir();
}

public interface Scanner {
    void digitalizar();
}
```

### Aplicação Em Cenários Reais
- Interfaces menores facilitam **reuso e testes unitários**
- Reduz impacto de mudanças em contratos
- Muito utilizado em APIs, frameworks e desenvolvimento orientado a interfaces em Java
