O **Open/Closed Principle (OCP)** estabelece que uma classe deve estar **aberta para extensão e fechada para modificação**, permitindo adicionar novos comportamentos sem alterar código já testado; em Java, isso é aplicado com **interfaces, classes abstratas e polimorfismo**, como em um sistema de pagamentos onde novos meios são adicionados sem mudar a lógica central, reduzindo risco de regressões e facilitando evolução contínua.

---
## Informações Adicionais Para Aprofundamento

### Exemplo De Violação
```java
public class PagamentoService {
    public void pagar(String tipo) {
        if (tipo.equals("PIX")) {}
        if (tipo.equals("CARTAO")) {}
    }
}
```

### Exemplo Aplicando OCP
```java
public interface Pagamento {
    void pagar();
}

public class Pix implements Pagamento {
    public void pagar() {}
}

public class Cartao implements Pagamento {
    public void pagar() {}
}
```

### Aplicação Em Cenários Reais
- Uso de **Strategy Pattern** para variar comportamento
- Redução de `if/else` e `switch` em regras de negócio
- Forte alinhamento com boas práticas de arquitetura e certificações Java
