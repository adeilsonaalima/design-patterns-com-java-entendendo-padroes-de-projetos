## Dependency Inversion Principle

O **Dependency Inversion Principle (DIP)** estabelece que módulos de alto nível não devem depender de módulos de baixo nível, mas ambos devem depender de **abstrações**, e que abstrações não devem depender de detalhes; em Java, isso reduz acoplamento ao usar **interfaces** e **injeção de dependência**, como em um serviço que usa um repositório abstrato sem conhecer sua implementação concreta, facilitando testes e evolução.

---
## Informações Adicionais Para Aprofundamento

### Exemplo De Violação
```java
public class PedidoService {
    private PedidoRepository repository = new PedidoRepository();
}
```

### Exemplo Aplicando DIP
```java
public interface PedidoRepository {
    void salvar();
}

public class PedidoService {
    private final PedidoRepository repository;

    public PedidoService(PedidoRepository repository) {
        this.repository = repository;
    }
}
```

### Aplicação Em Cenários Reais
- Base para **Injeção de Dependência** (Spring, CDI)
- Facilita **mocking** em testes unitários
- Reduz impacto de mudanças em infraestrutura e persistência
