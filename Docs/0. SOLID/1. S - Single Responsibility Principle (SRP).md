O **Single Responsibility Principle (SRP)** define que uma classe deve ter **apenas um motivo para mudar**, significando que ela deve assumir uma única responsabilidade bem definida no sistema; em Java, isso evita classes que concentram regra de negócio, persistência e apresentação, reduzindo acoplamento e facilitando manutenção, testes e evolução do código, assim como dividir funções em uma equipe torna mudanças mais previsíveis e controladas.

---
## Informações adicionais para aprofundamento

### Exemplo de violação
```java
public class PedidoService {
    public void calcularTotal() {}
    public void salvarPedido() {}
}
```

### Exemplo aplicando SRP
```java
public class Pedido {
    public void calcularTotal() {}
}

public class PedidoRepository {
    public void salvar(Pedido pedido) {}
}
```

### Aplicação em cenários reais
- Camadas **Service**, **Repository** e **Controller** com responsabilidades distintas
- Código mais aderente a testes unitários e manutenção evolutiva
- Princípio amplamente cobrado em certificações Java e base do **SOLID**
