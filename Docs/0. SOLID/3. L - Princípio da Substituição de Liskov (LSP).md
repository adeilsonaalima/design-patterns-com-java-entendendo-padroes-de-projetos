O **Princípio da Substituição de Liskov (LSP)** afirma que uma classe derivada deve poder substituir sua classe base sem alterar o comportamento esperado do sistema; em Java, isso significa respeitar contratos definidos por **interfaces ou classes abstratas**, garantindo que subclasses não enfraqueçam regras nem lancem exceções inesperadas, como trocar um cartão por outro no mesmo leitor sem causar falhas.

---
## Informações adicionais para aprofundamento

### Exemplo de violação
```java
public class Conta {
    public void sacar(double valor) {}
}

public class ContaSalario extends Conta {
    public void sacar(double valor) {
        throw new UnsupportedOperationException();
    }
}
```

### Exemplo aplicando LSP
```java
public interface Conta {
    void sacar(double valor);
}

public class ContaCorrente implements Conta {
    public void sacar(double valor) {}
}
```

### Aplicação em cenários reais
- Subclasses devem manter **pré-condições e pós-condições** da classe base
- Evita erros em coleções polimórficas (`List<Conta>`)
- Princípio essencial para herança correta e polimorfismo seguro em Java
